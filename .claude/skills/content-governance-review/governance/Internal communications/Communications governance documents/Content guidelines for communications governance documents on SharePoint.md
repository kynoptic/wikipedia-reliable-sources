# Content guidelines for communications governance documents on SharePoint

*Guidance on structuring and creating finalized communications governance documents for SharePoint to ensure consistency and clarity*

## Introduction

The purpose of this document is to provide a standardized template for creating and finalizing communications governance documents on SharePoint. This aims to ensure consistency, clarity, and effective communication across all communications governance hosted on SharePoint. The primary audience those involved in developing and maintaining governance documents.

This template applies to all finalized communications governance documents that will be hosted on SharePoint. It encompasses documents related to various aspects of communications governance, including style guides, workflows, and editorial guidelines.

The development of this template is based on the need for a standardized approach to governance documents, arising from the diverse and often fragmented ways in which these documents have been created and managed in the past. This template seeks to streamline the process, making it easier for users to create, find, and use governance documents. Input from the Communications Steering Committee has been incorporated to ensure the template meets the practical needs of the organization.

## Content guidelines

### Title

```html
Structure titles as <kbd>Content governance type</kbd> *for* <kbd>content type</kbd> *on* <kbd>platform</kbd>. If the document governs a content type across all platforms, or vice-versa, exclude that part of the title.
```
For example:

- Style guide for communications to users on all platforms
- Organization for communications governance documents on SharePoint
- Workflow for broadcast email communications
- Template for news articles on the HMS IT website
- Editorial guidelines for articles on the knowledge base

- 

- The filename must match the document title.

### Subtitle

The subtitle reiterates the title and offers more detail or a slightly different description to offer clarity.

When possible, use the terms typically used in content strategy (like *information architecture* instead of *organization*) to help ground the document and guide those in communications roles to better understand the scope of the document.

Don’t include a period at the end of the subtitle.

Incomplete sentences are OK.

### Table of contents

Include a table of contents if it will help the user navigate a longer document. Limit the number of headings displayed to keep the table of contents manageable. Exclude the table of contents from short documents. Exclude the introduction from the table of contents.

### Introduction

In paragraph prose, cover:

- **Purpose of the document** – Define the intent, objectives, and audience of this content governance document.
- **Scope and applicability** – Specify the boundaries within which this document applies.
- **Background** – Justify the governance with research, background information, and reasoning. Base the guidance on user needs and practical implementation for effectiveness. Include committee deliberations and decisions when relevant.
- Bold key information to make it easier to understand at a glance.

### Related governance

Link to related governance using the full document title.

### Summary

Provide an overview of the key points outlined in the document. In some cases, the main content is so short that a summary is unnecessary. Include visual aids to facilitate quick understanding.

### Main content

Structure the content to be clear and concise, with minimal overlap with other areas covering similar ground. Link to related governance in context when appropriate.
