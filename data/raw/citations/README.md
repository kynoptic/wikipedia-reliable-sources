# [`/data/raw/citations`]

## Purpose

Raw Wikipedia citation datasets pulled in from an external archive for
exploration. These are bulk enwiki citation/heading dumps, not generated by the
project's own fetching scripts.

## Contents

### Committed files

* **[`featured-articles.csv`](./featured-articles.csv)** ‚Äì Featured article list
  with `number, title, pageid, namespace, length, touched` columns.
* **[`good-articles.csv`](./good-articles.csv)** ‚Äì Good article list, same
  columns as the featured list.
* **`flow_[e084a770]_enwiki_2016-06-01_CS1_citations_Flow.zip`** ‚Äì Flow pipeline
  artifact bundle (a `.json5` flow definition plus an `artifact_*.data` blob).

### Untracked files (git-ignored ‚Äî see [`/.gitignore`](../../../.gitignore))

The bulk `*.tsv` dumps total ~5.7 GB and are kept out of git history. Copy them
back from the archive if missing.

* **`wikipedia-citations_enwiki.tsv`** (~298 MB) ‚Äì Citation identifiers per
  revision: `page_id, page_title, rev_id, timestamp, type, id` (e.g. `doi`).
* **`wikipedia-citations_enwiki 2.tsv`** (~393 MB) ‚Äì Variant of the above.
* **`wikipedia-citations_enwiki_2016-06-01_CS1_citations.tsv`** (~3.5 GB) ‚Äì
  Full CS1 citation records; final column is a JSON object per citation.
* **`wikipedia-citations_enwiki_20161101_headings.tsv`** (~1.1 GB) ‚Äì Article
  section headings: `page_id, page_title, page_ns, heading_level, heading_text`.
* **`wikipedia-citations_page2cat.tsv`** (~363 MB) ‚Äì Page-to-category mapping,
  one page per row with its categories tab-separated.

## Original processing pipeline

The Flow zip is an exported **Google Cloud Dataprep / Trifacta** flow
(`trifactaVersion 8.10.0`, exported 2022-01-01). It is the *original* analysis
these files fed into ‚Äî a predecessor to this repo's Python pipeline in
[`/core`](../../../core/). Reconstructed from the recipe steps, it ran four
chained recipes:

1. **`2016 citations`** ‚Äì Loads `enwiki_2016-06-01_CS1_citations.tsv(.bz2)`,
   parses each citation's JSON metadata blob, extracts the `URL`, and decomposes
   it into `host_URL`, `subdomain_URL`, `domain_URL`, and `suffix_URL` via
   regex. The template-name normalization table (`artifact_*.data`, e.g.
   `"Cite book " ‚Üí "cite book"`) canonicalizes citation types here.
2. **`Citations tagged FA & GA`** ‚Äì Joins each citing article's page id against
   `featured-articles.csv` and `good-articles.csv` (lookup on `pageid`) to derive
   `isFeaturedArticle` / `isGoodArticle` flags, and against the Perennial sources
   sheet for reliability status.
3. **`Pivot by host`** ‚Äì Groups by `(host_URL, subdomain_URL, domain_URL,
   suffix_URL)` and counts total citations, citations from FA, citations from GA,
   and distinct articles.
4. **`Pivot by domain`** ‚Äì Same aggregation at domain granularity.

Net result: per-host and per-domain citation-frequency tables tagged with
FA/GA usage and perennial-source reliability ‚Äî i.e. "which sources the best
articles cite most," the raw signal for ranking source reliability in the
Goggle. This repo's `core/` reimplements the same idea from live wikitext
rather than the static 2016 dump.

## Usage

Exploration data only ‚Äî treat as read-only. The large TSVs are git-ignored, so
clones will not include them.
